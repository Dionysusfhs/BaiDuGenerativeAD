# 目录结构

```
GenerativeAD/
├── data/               # 数据目录，存放sequence_data和ad_data
├── SASRec/            # SASRec模型实现
├── DCRec/             # DCRec模型实现
├── evaluate/          # 评估脚本和工具
├── checkpoint/        # 模型检查点保存目录
├── requirements.txt   # 项目依赖
└── README.md         # 项目文档
```

主要目录说明：
- `data/`: 存放原始数据集，包括用户序列数据和广告内容数据
- `SASRec/`: 自注意力序列推荐模型的实现代码
- `DCRec/`: 基于扩散模型推荐模型的实现代码
- `evaluate/`: 包含模型评估相关的代码和工具
- `checkpoint/`: 用于保存模型训练过程中的检查点文件
- `requirements.txt`: 列出项目所需的所有Python依赖包
- `README.md`: 项目说明文档

# 运行说明

## 环境配置
1. 创建并激活虚拟环境（推荐）
```bash
create -n GenerativeAD python=3.10
```

2. 安装依赖
```bash
pip install -r requirements.txt
```

## 数据准备
1. 将数据文件放置在正确位置：
```bash
mkdir  data
# 将 sequence_data 和 ad_data 文件放入 data/ 目录
```

## 模型训练

### SASRec 模型
在项目根目录下运行：
```bash
python SASRec/train.py
```

# 数据集格式说明

1. sequence_data 点击行为序列集，500000个用户，1分钟加载
* 每行两列, Tab分割
第一列为标识用户的id，第二列为交互的广告ID序列（序列元素通过空格分割) ）

| 字段 | 说明 | 数据样例 |
| - | - | - |
| 用户id | 用户的唯一标识 | 1 |
| 用户行为序列 | 每一位代表一个ad_id，空格分割 | 12132 12133 12182 15125 23133... |  

**注：** 假定序列长度为L，给定前L-1个元素形成的序列，推理第L个item即为本次比赛的目标。

2. ad_data 广告内容集，1307221个AD，十分钟加载

* 每行三列，Tab分割

| 字段 | 说明 | 数据样例 |
| - | - | - |
| ad_id | 广告的唯一标识符 | 12132 |
| content | 广告的描述信息，已tokenize成id形式。每个token使用逗号分割 | 21,32,12,98,101,200 | 
| ad_emb | 广告的预训练向量。向量的每个维度使用逗号分割 | 1.0231,1.7017,2.0594,-0.1968,-1.7533,-1.0083,-1.0436,-0.1506,-0.6201,1.1442,0.4997,0.7109,... |

**注：** 数据集保证序列中出现的所有广告ID均由对应的广告特征描述


此外，数据集附带了一份满足提交要求的样例代码`code.zip`，供选手理解应该如何提交结果。
